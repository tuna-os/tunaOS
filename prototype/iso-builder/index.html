In-browser pipeline · stages 1–2: read the image, find its kernel

The end state builds the whole ISO right here — no terminal, no app, no extra published artifacts: the source is the bootc image already on ghcr.io. Live today: stage 1 reads token → manifest index → platform manifest → config; stage 2 streams the image's tar+zstd layers (topmost first) through an in-page zstd decoder and tar walker until it locates the kernel and initramfs — no layer is ever held in memory. Needs the CORS shim (worker/cors-shim.js) because ghcr.io refuses browser reads.
