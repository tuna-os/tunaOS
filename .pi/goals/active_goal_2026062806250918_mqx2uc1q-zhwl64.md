- `build-yellowfin.yml` passes for all GNOME flavors: gnome, gnome50, gnome-hwe, gnome-nvidia, gnome50-hwe, gnome50-nvidia, gnome-nvidia-hwe

- Bluefin-derived improvements (new actions, testing patterns, prod promotion flows) are adopted

## Boundaries

**In scope:**
- Fix all failing GNOME image builds for albacore and yellowfin

- Port relevant CI/testing improvements from projectbluefin/bluefin and bluefin-lts
- Extend working GNOME pipeline to skipjack, bonito, and grouper

**Out of scope:**
- KDE, Cosmic, Niri, or other non-GNOME desktop variants
- bootc-installer GUI porting (separate goal)
- Non-GNOME ISO builds
- New GNOME features or package additions
- Kernel serial driver fix (CONFIG_SERIAL_8250=m in almalinux-bootc — separate follow-up)

## Constraints

- Must follow existing CI architecture (build-variant.yml → reusable-build-image.yml pipeline)
- ISO verification must not block existing non-GNOME builds
- Bluefin improvements are adopt-what-fits — no forced alignment
- Serial e2e is non-blocking: emits ::warning on failure, continues pipeline
